# Task: Deep Research on Open Source Community Building

## Research Objectives

Conduct comprehensive research on successful community building patterns specifically for **open source software projects**, with focus on:

### 1. Proven Community Building Frameworks
- What frameworks/methodologies exist for OSS community building?
- What are the stages of community growth (0→100, 100→1000, 1000→10000 users)?
- Critical success factors at each stage
- Common failure patterns to avoid

### 2. Case Studies: Successful OSS Community Building
Analyze 5-7 successful open source projects that built strong communities:
- How did they start? (first 100 users)
- What platforms did they use and in what sequence?
- Timeline from launch to mature community
- Key inflection points and what triggered them
- Specific tactics that worked (and which didn't)

**Prioritize projects similar to know-cli:**
- Developer tools
- CLI tools
- Infrastructure/workflow tools
- Recent launches (2020-2025 preferred for current tactics)

### 3. Fediverse/Decentralized Social Strategy
Deep dive on building communities through **Mastodon, Bluesky, and federated platforms**:
- How does community building differ on Fediverse vs. centralized platforms?
- Best practices for Mastodon engagement
- Bluesky-specific strategies (starter packs, feeds, custom algorithms)
- Instance selection strategy for developers
- Hashtag and discovery tactics
- Cross-posting strategies (Mastodon ↔ Bluesky)
- Success stories of projects that grew via Fediverse

### 4. Platform-Specific Tactics
For each platform, identify:
- **Discord**: Server structure, onboarding, engagement tactics
- **Reddit**: Karma building, subreddit strategy, anti-spam tactics
- **Hacker News**: Show HN best practices, comment engagement
- **GitHub**: Using Discussions, Issues, Stars as community tools
- **Mastodon**: Instance selection, hashtags, boosts, local vs. federated timeline
- **Bluesky**: Starter packs, custom feeds, community building features

### 5. Time-Efficient Community Building
Focus on **strategies optimized for 15-minute engagement blocks**:
- What high-leverage activities can be done in 15 minutes?
- Daily routines that maximize impact with minimal time
- Automation tools and tactics (without being spammy)
- How to build authenticity in short time windows

### 6. Pre-Launch vs. Post-Launch Strategies
- What's different about pre-launch community building?
- Optimal pre-launch duration (4 weeks? 8 weeks? 12 weeks?)
- How to maintain momentum post-launch
- Avoiding the "launch spike then crash" pattern

### 7. Target Community Identification
Identify **specific communities** to target for know-cli:

**Discord Servers:**
- AI agent development servers (LangChain, AutoGPT, CrewAI, etc.)
- CLI tool communities
- Developer workflow communities
- Graph database communities

**Reddit Subreddits:**
- AI development subreddits
- Programming and developer tool subreddits
- Open source communities

**Mastodon Instances:**
- Developer-focused instances
- Open source instances
- AI/ML focused instances
- Which instance to join as home base?

**Bluesky Communities:**
- Starter packs for developers
- Custom feeds for AI/dev tools
- Key accounts to follow/engage with

**GitHub Communities:**
- Related projects with active Discussions
- Organizations to engage with

For each target, provide:
- Direct links/names
- Member count / activity level
- Content focus and culture
- Engagement rules and norms
- Strategic rationale for targeting
