## Key GSD Patterns to Adopt

```xml
<task type="auto">
  <name>Task Title</name>
  <files>src/path/to/file.ts</files>
  <action>Specific HOW with library choices, avoid X because Y</action>
  <verify>curl command or test assertion</verify>
  <done>Observable acceptance criteria</done>
</task>
```

Checkpoint types: `checkpoint:human-verify` (90%), `checkpoint:decision` (9%), `checkpoint:human-action` (1% - rare)

User emphasized: "the full spec of how something is done in the task xml - minimize the guessing the agent has to do"
