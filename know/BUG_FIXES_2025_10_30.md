# Bug Fixes - 2025-10-30

## Summary

Fixed 4 bugs in the Know CLI tool by refactoring the codebase to use `dependency-rules.json` as the single source of truth for schema validation, rather than hardcoding values.

## Bugs Fixed

### Bug #1: Entity Type "operation" Referenced in Rules But Not Available

**Symptom:**
```bash
$ know add operation test-op '{"name":"Test"}'
Invalid entity type 'operation'. Valid types: action, asset, behavior, component...
```

But `dependency-rules.json` line 8 says:
```json
"component": ["operation"]
```

**Root Cause:**
`know_lib/entities.py` had hardcoded `VALID_ENTITY_TYPES` set that didn't include "operation"

**Fix:**
Refactored `EntityManager.__init__()` to load entity types dynamically:
```python
# Load from dependency-rules.json
self.VALID_ENTITY_TYPES = set(self.rules.get('entity_description', {}).keys())
```

**Files Changed:**
- `know_lib/entities.py` - Dynamic loading of entity types

---

### Bug #2: Build Order Reports Cycles When No Cycles Exist

**Symptom:**
```bash
$ know cycles
✓ No circular dependencies found

$ know build-order
Cannot determine build order (graph has cycles)
```

**Root Cause:**
`topological_sort()` in `dependencies.py` returned partial results when cycles prevented full processing, instead of detecting the problem.

**Fix:**
Added validation check:
```python
# If we didn't process all nodes, there are cycles
if len(result) != len(graph):
    return []
```

**Files Changed:**
- `know_lib/dependencies.py` line 307-309 - Cycle detection in topological sort

---

### Bug #3: "Disconnected Subgraphs" Warning for Valid Initial State

**Symptom:**
```bash
$ know health
⚠ 3 disconnected subgraphs
⚠ 11 warnings
[exits with error code 1]
```

For an intentionally minimal/incremental graph.

**Root Cause:**
Health check treated disconnected subgraphs as critical errors requiring exit code 1.

**Fix:**
Changed disconnected subgraphs to informational messages that don't cause failure:
```python
# Critical issues (cause failure)
has_critical_issues = not is_valid or cycles

# Only exit with error if there are critical issues
if has_critical_issues:
    sys.exit(1)
```

Disconnected subgraphs now show as:
```
ℹ 3 disconnected subgraphs (expected for initial/incremental graphs)
```

**Files Changed:**
- `know.py` lines 427-452 - Health check logic

---

### Bug #4: "Unexpected Fields" Warning for Useful Metadata

**Symptom:**
```bash
$ know add component card '{"name":"Card","path":"lib/Card.jsx"}'
$ know validate
⚠ Entity component:card has unexpected fields: path
```

**Root Cause:**
Validation had hardcoded allowed fields, didn't recognize useful metadata like `path`, `status`, `tags`.

**Fix:**
1. Added `allowed_metadata` to `dependency-rules.json`:
```json
"entity_note": {
  "schema": "Entities only have name & description keys.",
  "allowed_metadata": ["id", "type", "path", "status", "tags", "metadata", "notes"]
}
```

2. Refactored validators to load allowed metadata:
```python
entity_note = self.rules.get('entity_note', {})
self.allowed_metadata = set(entity_note.get('allowed_metadata', []))
```

**Files Changed:**
- `config/dependency-rules.json` line 182 - Added allowed_metadata field
- `know_lib/entities.py` lines 28-30 - Load allowed metadata
- `know_lib/validation.py` lines 35-37 - Load allowed metadata
- `know_lib/validation.py` line 148 - Use dynamic allowed_metadata

---

## Architecture Improvement

The core improvement is **eliminating hardcoded schema values**. Now:

- **Entity types** come from `entity_description` in rules
- **Allowed metadata** comes from `entity_note.allowed_metadata` in rules
- **Dependency relationships** come from `allowed_dependencies` in rules

This prevents schema drift between documentation and implementation.

### Before (Hardcoded):
```python
class EntityManager:
    VALID_ENTITY_TYPES = {
        'user', 'requirement', 'objective', ...  # Easy to miss updates
    }
```

### After (Dynamic):
```python
class EntityManager:
    def __init__(self, graph_manager, rules_path=None):
        with open(rules_path) as f:
            self.rules = json.load(f)
        self.VALID_ENTITY_TYPES = set(self.rules['entity_description'].keys())
```
